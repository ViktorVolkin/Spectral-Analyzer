# Анализатор Спектров Отражения (Spectral Analyzer)

Этот инструмент предназначен для автоматической обработки спектров отражения/пропускания. Он отфильтровывает аппаратный шум спектрометров, находит локальные интерференционные экстремумы (пики и долину) и вычисляет геометрические параметры макроструктуры: глубину провала (ΔR) и добротность/контрастность (Q).

## Как пользоваться

### Подготовка

Убедитесь, что у вас установлен Python 3.8+ и необходимые библиотеки.
Вы можете установить их командой:

```bash
pip install numpy pandas scipy matplotlib
```

### Формат данных

На вход подаются .csv файлы. Файл должен содержать как минимум две колонки:

Длина волны (λ, нм) ,Коэффициент отражения (R, %)

Заголовки колонок игнорируются, скрипт считывает данные строго по индексам столбцов.

## Запуск

Скрипт запускается через командную строку. Вы можете передать путь к конкретному файлу или к целой папке с файлами.

Обработка одного файла:

```bash
python main.py ./data/spectrum_01.csv
```

Пакетная обработка всей папки:

```bash
python main.py ./data/
```

### Дополнительные аргументы

Вы можете тонко настроить работу алгоритма с помощью флагов:

- --window : Задать жесткую ширину окна сглаживания в нанометрах (например, --window 15). По умолчанию стоит auto (умный автоподбор).

- --polyorder : Порядок полинома для фильтра Савитцкого-Голея. По умолчанию 3.

- --no-plot : Отключить генерацию и сохранение графиков (ускоряет работу при обработке тысяч файлов на выход будет только csv).

Пример:

```Bash
python main.py ./data/ --window 20 --polyorder 2 --no-plot
```

## Что на выходе?

- Все результаты сохраняются в автоматически создаваемую папку output/ рядом со скриптом.

- Внутри создаются подпапки для каждого обработанного файла с индивидуальными графиками (исходный спектр, сглаженный, производная, комбинированный отчет) и файлом result.json с точными координатами.

- В корне папки output/ генерируется файл summary.csv — это сводная таблица по всем файлам, идеально подходящая для передачи заказчику или вставки в Excel.

## Как это работает (Под капотом)

Программа не просто ищет максимальные/минимальные значения в массиве (что давало бы огромную погрешность из-за шума детектора). Она применяет комплексный математический конвейер.

### Шаг 1: Умный автоподбор окна и фильтрация

- Используется фильтр Савитцкого-Голея. В отличие от обычного скользящего среднего, он сохраняет физическую форму пиков и не срезает их высоту.

- Черновой проход: Скрипт сначала сканирует данные с очень широким окном (60 нм), чтобы определить макро-расстояние между интерференционными пиками.

- Чистовой проход: На основе найденного расстояния алгоритм вычисляет идеальное окно (ровно половина расстояния от пика до долины). Это гарантирует максимальное сглаживание шума без искажения физической глубины ΔR.

### Шаг 2: Поиск экстремумов (Суб-пиксельная точность)

Чтобы найти вершины и долину, скрипт берет первую производную сглаженного спектра (dR/dλ):

- Пики и долины находятся в точках, где производная пересекает ось нуля.

- Адаптивная полировка: Для спектров с высоким разрешением (более 400-500 точек) производная дополнительно пропускается через легкий uniform_filter1d. Это устраняет математическую ребристость (алиасинг) и предотвращает ложные срабатывания на микро-шумах.

- Интерполяция: Точная координата нуля ищется с помощью линейной интерполяции между двумя соседними точками измерения. Это позволяет локализовать пик с точностью выше, чем шаг самого спектрометра.

### Шаг 3: Геометрический расчет ΔR и Q

Как только найдены Левый Пик (P1), Правый Пик (P2) и Долина (V) между ними:

- Выстраивается базовая линия (хорда) — прямая, соединяющая вершины P1 и P2.

- ΔR (Глубина): Рассчитывается как перпендикуляр от точки долины V до пересечения с базовой линией. Это позволяет честно оценить глубину провала даже на спектрах с сильным общим наклоном.

- Q (Отношение): Рассчитывается как процентное отношение глубины ΔR к абсолютной высоте первого (левого) пика.
